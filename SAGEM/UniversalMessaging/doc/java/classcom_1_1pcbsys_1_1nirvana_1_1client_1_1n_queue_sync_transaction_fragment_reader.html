Detailed Description

This reader provides a synchronous queue reader model.

Where the client can pop() events from the queue and then decide either to commit the event, ie. successfully processed it or to rollback, ie unable to process this event. This reader behaves exactly like the nQueueSyncTransactionReader with the exception that it understands fragmented events and will only return an event when all fragments are received. When the window size is reached and the client has yet to call commit or rollback the server will raise an exception on the pop() command until the client has called either method.

Deprecated:
Message fragment will not be supported in UM hence this class is deprecated as of version 10.7 and will be removed in the subsequent release
