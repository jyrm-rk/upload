* Command Central Security Roles


* DO NOT MODIFY PERMISSIONS SECTION
[permissions]
permission:allow=canwrite,canexecute,canread,jmx

* Available permissions:
* canread – can view everything
* canwrite – can create, update, delete anything
* canexecute - can execute lifecycle and administration operations (e.g., start)
* jmx:com.softwareag:passman:canexecute:* - can retrieve passwords
*
* To remove a role or assignment, delete the line.

[roles]
* Define one role per line. To specify all permissions, use *.
* role:<role_name>=<permission>,<permission>[<permission>...]

* superadmin role has full control
role:superadmin=*

* operator role can view everything and execute operations, but cannot edit anything
role:operator=canread,canexecute

* viewer role has readonly access to everything
role:viewer=canread

role:passman=jmx:com.softwareag:passman:canexecute:*

[groups]
* Group names are case sensitive.
*
* Assign one or more roles to one internal group per line. 
* group:"<group_name>"=<role>,<role>[<role>...]
* Examples
* group:"Operators"=operator
* group:"Developers"=viewer

* Administrators group has full control
group:"Administrators"=superadmin,passman


[users]
* Assign one or more roles to one internal user per line. 
* user:"<user_name>"=<role>,<role>[<role>...]
